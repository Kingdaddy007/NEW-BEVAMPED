# Mistakes to Avoid

- **Do not overwrite existing approved assets without checking.** In the past, we overwrote room names and descriptions for videos the user already liked (Bath, Transit) to force a "Tropical" theme everywhere. Always isolate the "missing gap" and only update what is requested.
- **Avoid giant, overlapping typography on videos.** The previous `h1` tags spanning across the videos made the site look cheap and obscured the assets. Use restrained, gallery-style lockups.
- **Do not ignore the narrative context.** The user wants smooth, cinematic transitions (like the expanding square) rather than abrupt jump cuts between sections.
- **Avoid inline HTML `onload` attributes for sibling visibility changes.** If an image is cached, the browser fires `onload` immediately when parsing the `<img>` tag before the next sibling is added to the DOM, returning `null` for `this.nextElementSibling` and causing placeholder cards to leak. Always handle load transitions dynamically in JavaScript using `.complete` checks and load listeners.
- **Initialization Order in Complex GSAP Files:** Do not initialize DOM node variables (like gsap.utils.toArray('.my-cards')) after a ScrollTrigger has already tried to use that variable's .length property. During CSS/JS rewrites, verify initialization order or you will trigger a ReferenceError that breaks all rendering.
- **Missing DOM Declarations in Timelines:** Ensure every element referenced in a `masterTl.call()` or animation block is properly defined (e.g., `actIForeground`). A missing variable throws a silent ReferenceError that completely halts the GSAP timeline and breaks subsequent transitions (like the SVG blinds mask).
- **Dark Text on Raw Cinematic Video:** Never use dark/thin text over raw video backgrounds unless there is a strong, blurred, or solid background plate. The shadows in the video will render the text invisible. Default to Ivory text with drop shadows for high-end legibility over video.
- **Do NOT directly copy motion references verbatim.** When provided with a reference site (e.g., Good Fella), do not literally copy their exact interaction mechanics (like a specific scrolling dot wave). Extract the underlying principle (e.g., "scroll-linked typography transformation") and invent a completely unique, premium Awwwards-style implementation (like Hover Blur Focus) to ensure originality.
- **Always Verify Code Post-Edit:** Never assert an edit is complete without verifying side-effects. Injected HTML without closing tags can break scroll containers, and mismatched classes can cause hero animations to fail silently.
- **Do not surrender art direction immediately on first critique:** When a user expresses discomfort with a design choice (e.g., an edgy letterform like Syne), do not instantly scramble to scrap the entire system. Instead, pause, rely on the 6 Meta-Cognitive Safeguards (Anti-Comfort), evaluate the brand psychology, and present a deeply reasoned, systemic alternative rather than a menu of random options.
- **Do not default to standard Hex codes blindly:** Avoid generic shades (like `#1A1F36` Navy) for luxury brands. Always test the *depth* of the color against the brand's physical equivalents (like Obsidian vs Plaster) to prevent accidentally rendering a "SaaS dashboard" aesthetic.

- **Do not abruptly invert section colors**: Avoid changing the background color of a section (e.g. Black to White) at the bottom of a page unless it deliberately bridges into the footer or the specific next section. Unnecessary contrast flips break the visual flow.

- **Absolute Boundary Clipping**: Setting absolute boundary text sizes too large (e.g., clamp 3rem to 6rem with line-height: 1) on dark-to-light section transitions causes the text ascenders to overlap invisible areas and clip horizontally. Always add interior container padding to contain text ascenders.
- **Navigation Link Styling Inheritance**: Assigning anchor (\<a>\) tags to logo containers without explicitly resetting \color\ and \	ext-decoration\ causes unwanted browser-default link styling (e.g., standard blue) to cascade over carefully planned color palettes.
